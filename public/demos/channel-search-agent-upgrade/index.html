频道搜索 Agent 化改造

这个静态 Demo 只聚焦搜索框本体。页面其他部件故意弱化，用于比较“经典态、URL 识别态、自然语言展开态、混合输入态”的视觉和交互语义。

视觉方向

方案 A

平衡型

保守升级

最贴近现有 Nox 搜索页，只在展开态加入少量 AI 材质与分层，适合先做 v1。

方案 B

AI 材质型

推荐方向

搜索框外保持不动，展开后通过独立材质、柔和光感、层叠卡片制造“进入智能模式”的惊喜。

方案 C

强惊喜型

上限探索

更明显的 AI 发光边界与高对比层次，适合探索上限，但风险是和外部页面拉开过大。

方案 D

查询编辑器型

Gmail / 旧版 Nox

最强调“查询片段”语义。关键词 tag 更轻、更扁、更像输入内容被固化，不主动制造很强的 AI 面板感。

方案 E

专业工作台型

LinkedIn / Klear / Linear

更偏专业搜索工具。边界更稳、对比更清楚、解释性更强，适合把“AI 是能力层”这件事做得可信但不喧宾夺主。

方案 F

轻助手摘要型

Modash / ChatGPT Search

保留搜索工具骨架，但在搜索框内增加更柔和的任务摘要、理解反馈和轻提示，让用户感知“系统在协助我搜索”。

路径 A：经典关键词

输入关键词 → 固化为 tag → 保持经典态搜索 → 用户主动补一句自然语言时才升级为 Agent 展开态。

路径 B：纯 URL

输入 URL → 进入短暂识别态 → 能精确识别就走原有路径；识别失败或平台不匹配时进入修正态。

路径 C：混合输入

先解释系统如何理解锚点、硬条件、软偏好 → 先搜一轮 → 只在影响很大时再问一个问题。
