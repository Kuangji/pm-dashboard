# 频道搜索 Agent 化改造 · 调研简报

## Phase 0：决策锚定

| 维度 | 结论 |
|---|---|
| 决策目标 | 为 NoxInfluencer 频道搜索从“关键词 + URL + 筛选器”升级为“Agent 化自然语言搜索工作台”提供功能边界、交互骨架、视觉策略与 VDR 输入 |
| 功能边界 | 聚焦 `频道搜索`，包含搜索框、追问、条件透明、结果联动、零结果修正、历史会话；不包含视频搜索、收藏/联系工作流、底层索引重构 |
| 竞品候选 | 直接竞品：Modash、HypeAuditor、Upfluence、Klear；邻域对照：LinkedIn Recruiter / AI-Assisted Search |
| 采集模式 | 全部为 `[自动]`，优先官方产品页、官方帮助中心、官方博客/帮助文档 |
| 产出用途 | ✅ 直接写入 VDR：`input_v1.md`、`spec_v1.md`、`design_v1.md`、`prd_v1.md` |

## 用户已确认的核心前提

- 新形态不是抽屉，不是独立 chatbox，而是 `内联扩展搜索框`
- 自然语言首输和后续追问都必须发生在 `搜索框内部`
- 用户注意力始终留在搜索框
- 搜索框外部页面骨架尽量保持原状
- 惊喜感来自 `展开的搜索框内部组件` 的 AI 化材质与层次变化
- 默认搜索节奏是 `先搜后追问`
- 目标用户为 `全量兼容`，不能牺牲老用户关键词/URL 搜索效率

## 关键待回答问题

1. 市场上哪些产品已经把 AI 真正做进搜索主流程，而不是外挂助手？
2. 直接竞品中，哪类产品是“自然语言 + filters”，哪类是“独立 scout/brief”？
3. 搜索框内追问、结果区稳定、条件透明，这三件事如何共存？
4. 哪些视觉策略适合“局部异化”，不会把整页变成聊天产品？
