# 调研简报：关键词搜索功能竞品调研

## Phase 0 确认项

| 项目 | 内容 |
|------|------|
| 竞品 | 仟传 Tarsight（强登录验证，手动截图模式）|
| 功能域 | 达人/内容**关键词搜索** + **数据聚合分析** |
| 决策目标 | NoxInfluencer vs 竞品A 的差距：① 同一关键词下的数据量差异 ② Badcase 统计量差异 |
| 对比关键词 | Dreame |
| 产出用途 | 独立调研报告（暂不对接 VDR）|
| 采集模式 | 手动截图辅助（用户操作竞品 + 提供截图，AI 引导并结构化记录）|

## 决策背景

NoxInfluencer 的关键词搜索功能目前存在以下待评估问题：
- 数据覆盖量是否达到竞品水平？
- 搜索结果的相关性（Badcase 率）是否有明显差距？

本次调研通过与仟传 Tarsight 的横向对比，量化上述差距，为功能优化提供数据支撑。

## 截图会话协议

1. **每次只给一个操作指令**，等收到截图后再给下一步
2. **每张截图**：AI 解读 → 记录到 `01_raw_data/竞品A_raw.md` → 追问/下一步
3. **无法截图的场景**记录为 `❓ 未能观察`
4. **关键数据**（结果总数、每页数量等）用加粗标注

## 对比框架

| 对比维度 | 评估方式 |
|---------|---------|
| 数据量 | 同词搜索结果总数（Tarsight vs NoxInfluencer）|
| Badcase 率 | 抽样前 20 条，统计明显不相关条目数量 |
| 聚合视图 | 是否提供趋势图、品类分布、数据统计面板 |
| 交互体验 | 搜索入口、结果展示、筛选/排序能力 |
