# 执行记录

## 1. 环境与页面校验

- 执行时间：2026-04-08T09:03:18.761Z
- 使用方式：已登录 Playwright 浏览器上下文
- 页面：YouTube 频道搜索页
- 页面校验：
  - 已确认不是匿名态或演示态
  - 已确认 `100 / Page`
  - 已确认排序 radio 映射包含：
    - `followers`
    - `estExposure`
    - 默认排序（真实接口为不传 `sortField`）

## 2. 真实接口与口径

- 结果列表接口：`/ws/youtube/star/search`
- 频道视频补充接口：`/ws/youtube/star/search/resultVideo`
- 关键词集：`consumer electronic, outfit, camping, AI, game, lifestyle`
- 结果总数：
  - 实际使用 `retData`
  - `totalSize` 在大结果集上会被 cap 到 `10000`，只可当作窗口上限，不可当作真实总数

## 3. 默认排序口径

- 默认排序必须 **省略 `sortField` 参数**
- 若传 `sortField=null`，结果会退化成与 `followers` 相同，不可作为基线

## 4. 判定原则

- `fit`：频道主题与近期内容都能直接支持该关键词被召回
- `adjacent`：存在弱相关/场景相关/品牌相关，但不是强相关优质召回
- `clear_badcase`：频道整体主题明显不符，只因排序字段靠前而进入 Top100
- 主报告展示原则：先给判定规则，再给默认排序对照，最后给 clear badcase 证据卡
