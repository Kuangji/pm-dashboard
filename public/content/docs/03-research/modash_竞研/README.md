## 项目概览

本项目围绕 **Modash** 做平台级全景研究，重点回答四类问题：

1. **Modash 是什么**：它在官方口径里到底是“数据库”“营销平台”还是“创作者运营系统”？
2. **为什么有人买**：真实用户买单的核心原因是数据、效率、工作流还是价格？
3. **它强在哪**：相较 NoxInfluencer 及头部平台，Modash 的优势到底落在发现、管理、追踪还是支付？
4. **它相对 Nox 处于什么位置**：哪些是 Nox 应该正视的差距，哪些只是产品包装与目标客群差异？

本项目不默认产出 Nox roadmap，但会保留可复用为策略输入的洞察。

## 首版结论速览

- **Modash 的核心不是“数据最多”，而是把 Discovery、Outreach、Track、Pay 串成一个轻量工作流。**
- **它明显偏 DTC / Shopify / always-on creator program 场景**，不走重型企业咨询/实施路线。
- **社区对它的高评价主要来自“好上手、搜索和筛选强、支持响应快、替代 Excel/Notion/多工具拼接”。**
- **社区对它的质疑主要集中在“数据库口径偏宽、仍需人工 vetting、某些 tracking/reporting/payment 边角不够成熟、对小团队价格不算低”。**
- **相对 Nox，Modash 更像 workflow product；相对 CreatorIQ/GRIN/Upfluence，Modash 更轻、更快、更透明；相对 HypeAuditor，Modash 更偏执行闭环而非纯分析/稽核。**

## 口径说明

- **P0 高置信**：Modash 官网、帮助中心、定价/功能页
- **P1 中置信**：G2、Reddit 公开讨论
- **P2 辅助**：工作区历史扫描资料、行业榜单与横评

本项目明确区分三类内容：

- **事实**：能回溯到具体来源
- **信号**：来自评论/社区的重复观点
- **判断**：基于事实与信号的归纳，不当作官网事实表述
