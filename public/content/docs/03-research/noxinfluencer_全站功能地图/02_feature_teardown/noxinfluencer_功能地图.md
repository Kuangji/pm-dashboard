# NoxInfluencer 功能地图

## 总览

基于 2026-04-21 的真实浏览器实勘，Nox 当前站点更适合拆成四层，而不是把所有页面平铺成一个长列表：

1. **公开获客层**：负责解释产品、转化用户、提供公开工具与内容资产
2. **发现与情报层**：负责找达人、找商品、看品牌、看竞品
3. **运营执行层**：负责沉淀对象、触达、沟通、追踪、支付和商品协同
4. **平台治理与服务层**：负责配额、权限、安全、账单、通知和 API Key

这种结构说明：**Nox 的站点本质上已经是一个多阶段营销工作台，而不是一个单点搜索站。**

## 5. 最终判断

Nox 的全站功能地图不是“搜索 + 若干工具”这种松散拼接，而更接近：

> **公开获客入口 + 情报发现引擎 + 执行工作台 + 平台治理底座**

如果只看首页，容易把它理解成“数据/工具站”；如果只看工作台，又容易漏掉它的公开获客层与服务层。  
后续无论做竞品拆解还是内部需求设计，都应默认用这四层结构来定位模块。
