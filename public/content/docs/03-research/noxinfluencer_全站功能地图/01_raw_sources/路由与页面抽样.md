# 路由与页面抽样

## 1. 调研方式与会话重置说明

### 1.1 调研方式

本轮采用“**路由清单优先 + 代表页抽样校验**”的方法：

1. 用真实浏览器访问 `https://cn.noxinfluencer.com/`
2. 清空 cookies / localStorage / sessionStorage 后重新从首页开始
3. 先确认首页出现登录/注册入口，再使用广告主账号登录
4. 在登录后页面提取前端路由分组
5. 对公开层、登录后层、治理层的代表页分别做抽样

### 1.2 会话与账号

- **会话处理**：已清 cookies / storage 后重新访问
- **账号类型**：广告主账号（brand）
- **访问日期**：`2026-04-21`
- **操作原则**：只读调研，不主动创建、编辑、删除业务数据

### 1.3 事实来源标签

| 标签 | 含义 |
|------|------|
| `公开页可见` | 无需登录即可访问 |
| `登录后可见` | 登录后才能稳定访问 |
| `路由存在但需谨慎推断` | 前端路由可见，但未必代表稳定能力 |
| `当前不可用/被拦截` | 当前访问结果不能直接当作稳定能力 |

## 6. 本轮证据的关键判断

1. **Nox 的真实结构不是“公开官网 + 搜索页”这么简单**，后段执行与治理模块已经非常完整。
2. **搜索与品牌情报是发现层双入口**：一个从达人/商品出发，一个从品牌/竞品出发。
3. **运营执行层已经形成闭环**：收藏夹、CRM、邮件、消息、监控、支付、商品中心不是孤立工具。
4. **治理层是单独一层**：数据用量、权限、安全、通知、Skills 与业务页有明确分工。
5. **路由存在不代表稳定产品能力**，尤其是 `ai-assistant` 与 `affiliation` 相关入口。
