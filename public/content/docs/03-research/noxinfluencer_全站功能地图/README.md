## 项目概览

本项目的目标不是再做一轮零散功能调研，而是把 **NoxInfluencer 当前站点实际存在的主要功能模块、层级关系、能力边界与证据口径** 一次性整理成后续可复用的基础上下文。

本轮固定采用：

1. **路由清单优先**：先看前端实际暴露出的路由分组，而不是只凭首页导航猜结构。
2. **代表页抽样校验**：每个一级模块至少抽一个真实页面，记录标题、正文片段、最终 URL。
3. **能力认知落盘**：把“页面存在”升级成“它解决什么问题、处于链路哪里、容易和谁混淆”。

本项目默认服务于后续三类工作：

1. 站级调研与模块评价
2. 竞品对照与能力缺口判断
3. 新需求设计前的模块定位与事实核验

## 首版结论速览

- **Nox 不是单纯的“达人搜索工具”**，而是由公开获客层、发现与情报层、运营执行层、平台治理与服务层组成的复合平台。
- **站点真正的主链路已经存在**：找达人/商品/品牌情报 -> 建档与沉淀 -> 批量触达 -> 实时沟通 -> 监控发布效果 -> 支付结算 -> 配额/权限/治理。
- **Nox 的风险不是“模块不够多”，而是模块分布较散**，新读者容易只看到搜索或榜单，而漏掉 CRM、消息、支付、配额、Skills 等后段能力。
- **部分前端路由存在但不能直接当作稳定能力**，例如 `ai-assistant` 当前呈现 blocked page，`/affiliation/list/member` 当前返回 `Page Not Found`。
- **`/search/youtube/channel` 是高优先级真实入口**，虽然它没有在本轮抓到的 Nuxt 路由分组中完整显式列出，但首页 CTA 与登录后工作台都实际指向它。

## 口径说明

本项目区分四类证据标签：

- **公开页可见**：无需登录即可通过官网访问并读取
- **登录后可见**：需要广告主账号登录后才能稳定看到
- **路由存在但需谨慎推断**：前端路由表可见，但未必代表稳定业务能力
- **当前不可用/被拦截**：本轮访问时不可直接使用，不纳入稳定能力

本项目同时区分三类表述：

- **事实**：可回溯到页面标题、正文片段或路由
- **判断**：基于多页证据归纳出的结构理解
- **边界提醒**：用于防止后续误把两个模块当成一个模块
