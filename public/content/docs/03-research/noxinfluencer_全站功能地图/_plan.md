# NoxInfluencer 全站功能地图计划

## 项目背景与目标

**背景**：虽然工作区里已经积累了不少 Nox 单模块需求与竞品资料，但缺少一份能覆盖全站主要模块、可作为所有后续研究共用入口的站级公共上下文。

**目标**：输出一套“可追溯、可复用、可继续迭代”的 Nox 全站认知基建，而不是只写一个页面列表。

**固定研究问题**：

1. Nox 当前站点可以稳定分成哪些层？
2. 每一层的主模块分别解决什么问题？
3. 哪些模块属于前台获客，哪些属于后台工作台，哪些属于治理与服务？
4. 后续做竞品对比或需求设计时，最容易遗漏或混淆哪些边界？

## 可信度备注

- **高置信**：公开导航、登录后真实页面、标题、正文片段、最终 URL
- **中置信**：基于多页证据归纳出的层级关系与主链路
- **谨慎使用**：只在前端路由中出现、但页面未能稳定打开的能力

## 已回答 / 未完全回答

### 已回答

- Nox 当前站点可稳定分为四层，而不是单一搜索产品
- 运营执行层已经包含 CRM、邮件、消息、监控、支付等完整后段模块
- 数据用量、权限、安全、Skills 构成独立的平台治理与服务层
- 公开获客层与登录工作台层需要分开读，不能混写

### 未完全回答

- `affiliation` 命名空间当前在真实业务中的产品角色
- `ai-assistant` 的稳定产品边界与上线状态
- 某些重复路由形态（如 `/payment/list/list`）是历史兼容还是框架产物

> 这些未完全回答项不会阻塞当前首版公共上下文，但适合后续作为 v1.1 补充。
