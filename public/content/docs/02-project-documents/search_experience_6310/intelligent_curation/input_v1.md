# 智能精选 · Input v1

> 归属版本：`v6.3.10`  
> 前置依赖：`v6.3.9`

## 背景

`v6.3.9` 已经解决了标准搜索自然语言输入和单频道相似网红。剩余的智能精选能力仍然变数较大，需要独立成下一版继续打磨。

在重新梳理后，`智能精选` 不再被定义为“代替经典搜索的一套 AI 助手”，而是经典搜索之后的下一阶段：

- 经典搜索负责产出候选频道列表
- 智能精选负责对这份列表做进一步 deep research 与全景数据精筛

## 目标

建立一个独立的 `智能精选` 工作台：

- 先锁定一份 `冻结的输入列表快照`
- 同时引用生成这份快照时的 `当前搜索条件集`
- 用户用自然语言 `精选策略指令` 在这份列表上继续精筛
- 系统返回可持续消费的当前轮次结果
- 用户通过 `采纳 / 排除`、`继续筛 / 重新筛` 持续塑造 shortlist

## 本期包含

- 智能精选工作台
- 输入列表快照模型
- 轮次结果消费
- 精选策略指令
- session 持久化与独立化
- `标准搜索 -> 智能精选` 桥接

## 本期不包含

- 其它入口到智能精选的桥接
- 低召回 / 失败或超时完整分支
- 收藏/营销计划深度写入
- 完整策略编辑器
- 树状历史 / 全量轮次 lineage 图

## 成功标准

- 用户能理解智能精选不是标准搜索结果页
- 用户能理解桥接带来的是一份可追溯的结果快照
- session 之间彼此独立
