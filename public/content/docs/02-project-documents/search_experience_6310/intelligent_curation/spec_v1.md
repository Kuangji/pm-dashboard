# 智能精选 · Spec v1

> 归属版本：`v6.3.10`  
> 前置依赖：`v6.3.9`

## 一、输入起手

`v6.3.10` 中，智能精选不再作为独立全库搜索入口存在。

本版本唯一锁定的起手是：

- `标准搜索结果页 -> 智能精选`

桥接带入的不是“几条 seed 字段”，而是：

- 一份 `冻结的频道列表快照 / 引用`
- 生成该快照时的 `当前搜索条件集`

本版本桥接范围仅限 `标准搜索 -> 智能精选`。更广义的“任何频道列表 -> 智能精选”作为后续扩展方向保留。

## 二、工作台结构

顶部顺序固定为：

1. `当前输入快照`
2. `当前精选策略指令`
3. `当前 session 集合摘要`
4. `当前轮次结果`

其中：

- `当前输入快照` 负责说明来源、规模、生成时间与来源条件摘要
- `精选策略指令` 是当前轮次的主输入对象
- 工作台是“列表后处理”界面，而不是搜索框替代品

## 三、精选策略指令

前台采用“半透明模式”：

- 用户主入口是一句自然语言策略指令
- 系统内部会解析出结构化策略
- 前台默认只展示轻量 `系统理解摘要`
- 不在第一版开放完整策略编辑器

策略输入规则：

- 默认语义是 `继续筛`
- 即：下一条策略默认基于 `当前列表` 继续收窄
- 若用户不认可当前轮次结果，应显式触发 `重新筛`
- `重新筛` 会回到 `本轮直接基底` 重新生成，而不是继续在当前结果上收窄

## 四、基底选择器

基底选择器服务于“下一条策略将作用于哪份列表”，不负责切换当前结果浏览对象。

第一版只开放：

- `当前列表`
- `原始快照`
- `最近 3 轮结果`

规则：

- 默认基底 = `当前列表`
- 切换基底不会立即改动当前结果区
- 用户提交新策略后，才基于所选基底生成新一轮结果
- 新一轮生成后，默认基底回到新的 `当前列表`

## 五、结果消费

- 当前轮次卡片显式动作只保留：
  - `采纳`
  - `排除`
- 生效后条目立即从当前列表移除
- `已采纳` 作为正向偏好
- `已排除` 作为硬过滤
- `待定` 不再是正式桶
- 轻量 diff 只需表达：
  - 相对本轮直接基底的变化
  - 相对原始快照的累计收敛程度

## 六、Session

- 每个 session 至少保留：
  - 输入快照 / 来源条件
  - 当前策略指令
  - 已采纳 / 已排除
  - 当前轮次
  - 进度状态
- session 之间完全隔离
- 刷新后默认恢复最近活动的未归档 session
- 基于 `当前列表 / 原始快照 / 最近旧轮次` 发起的新策略，仍属于同一个 session 的新一轮
- 第一版不做树图，不开放全量历史轮次选择

## 七、桥接

标准搜索桥接规则：

- 只出现在标准搜索结果页
- 带入的是标准搜索当前结果快照与来源条件
- 会创建新的智能精选 session
- 不改写标准搜索原状态
- 桥接文案应强调“对这份结果做智能精选”，而不是“带条件去 AI 搜索”

## 八、验收标准

- 智能精选能独立表达输入快照、策略、轮次和 session
- 从标准搜索桥接时能明确看出输入来自一份可追溯结果快照
- 用户能清楚区分 `继续筛` 与 `重新筛`
- `v6.3.10` 文档不再重复定义 `v6.3.9` 主体能力

## 九、回看点

- `继续筛 / 重新筛` 的 UI 位置与文案，需在 Demo 出来后重新确认是否需要调整
