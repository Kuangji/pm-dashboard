# 智能精选 · Design v1

> 归属版本：`v6.3.10`  
> 前置依赖：`v6.3.9`

## 一、页面气质

- 它应该像独立工作台
- 但不做成聊天产品
- 它是频道列表之后的下一阶段，不覆盖标准搜索
- 它处理的是“输入列表快照”，不是“再搜一次”

## 二、工作台

### 2.1 顶部结构

- 输入快照卡
- 精选策略输入区
- session 集合摘要
- 当前轮次结果

其中：

- 输入快照卡需要明确区分：
  - 来源列表
  - 快照规模
  - 生成时间
  - 来源搜索条件摘要
- 精选策略输入区默认是“继续筛”主路径
- 不把策略编辑器完全摊开，只展示轻量系统理解摘要

### 2.2 session 侧栏

- 左侧显示最近使用 session
- 支持改名、归档、恢复
- 刷新后可恢复最近活动 session

### 2.3 基底选择器

- 放在策略输入区内
- 语义是“下一条策略将基于哪份列表执行”
- 默认选中 `当前列表`
- 第一版只开放：
  - 当前列表
  - 原始快照
  - 最近 3 轮结果
- 不做树图，不做全量历史轮次浏览

## 三、桥接

- `标准搜索 -> 智能精选` 是本版本唯一锁定的桥接
- 桥接入口挂在标准搜索结果页
- 视觉上是次级 CTA，不抢标准搜索主路径
- 桥接后进入的是“基于结果快照的智能精选工作台”，不是独立 AI 搜索首页

## 四、消费路径

- 执行中
- 一批结果
- 采纳 / 排除
- 继续筛或重新筛

其中：

- `继续筛`：
  - 默认主路径
  - 基于当前结果继续收窄
- `重新筛`：
  - 明确次级动作
  - 基于本轮直接基底重新生成一版
  - 需要在 UI 上与“继续筛”显式区分

## 五、结果对比

- 第一版只做轻量差异提示
- 主对比对象是 `本轮直接基底`
- 次级累计信息表达相对 `原始快照` 的收敛程度
- 不做重型 diff 视图

## 六、版本边界

- 智能精选整体延后到 `v6.3.10`
- `v6.3.9` 不承载任何智能精选产品入口

## 七、回看点

- `继续筛 / 重新筛` 的入口位置和提示强度，需要在 Demo 出来后重新确认
