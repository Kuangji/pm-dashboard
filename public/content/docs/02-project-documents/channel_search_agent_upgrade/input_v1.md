## 一、背景与动机

### 1.1 当前搜索模块的结构性问题

NoxInfluencer 现有频道搜索主流程仍然以：

- 关键词
- 频道 URL
- 一组显式筛选器

为核心输入模式。

在当前线上页面中，`AI助手` 已经存在，但它以 `搜索框右侧独立入口 + 抽屉/弹层` 的方式出现，输入、追问和结果不在同一主控件内完成，带来三个核心问题：

1. **像外挂，不像主流程**
   - 用户进入 AI 后，注意力离开搜索框
   - AI 像附加工具，而不是搜索本体升级
2. **状态分裂**
   - 用户难以判断当前结果到底由关键词、显式筛选器还是 AI 建议驱动
3. **新手门槛仍高**
   - 不会写条件的用户仍要在“搜索框 / AI助手 / 筛选器”之间切换

### 1.2 为什么现在要做

- 已有真实用户输入数据表明，大量搜索需求本质上是 `描述性自然语言问询`
- `02_Research_Analysis/AI_Search_User_Study/` 中的样本显示：
  - 自然语言需求占比显著高于纯关键词
  - 输入常同时包含地区、平台、粉丝量、内容类别、排除条件、活跃度等复合要求
- 市场上已经出现更成熟的搜索形态：
  - `Modash`：自然语言直搜 + filters 收口
  - `LinkedIn Recruiter`：AI-Assisted Search 与 current filters 同 session 协作
  - `HypeAuditor`：AI Search 与独立 AI Scout 双路线并存

结论：Nox 的下一步不是“把 AI 抽屉做得更强”，而是“把搜索框本体升级”。

## 二、目标用户

### 2.1 主要用户

- 不熟悉搜索语法或筛选体系的品牌方、投放运营、商务
- 已经熟悉关键词 / URL / 筛选器的老用户

### 2.2 用户分层目标

本次不是只服务新手，也不是只给专家加一个高级模式，而是：

- **新手**：可以直接用自然语言起搜
- **老用户**：仍可保持关键词 / URL / 手动筛选效率
- **全量用户**：都在同一个搜索框容器内完成输入和 refine

---

## 三、产品目标

### 3.1 核心目标

把频道搜索升级为 `原生搜索页 + 轻助手增强` 的 Agent 搜索工作台，使用户可以：

1. 用描述性自然语言直接发起搜索
2. 在首轮结果基础上继续 refine
3. 始终看见系统如何把需求映射成结构化条件
4. 不离开搜索框就完成后续追问与补充

### 3.2 成功标准

- 用户能用自然语言直接起搜，而不是被迫猜关键词
- 首轮结果尽快出现，不被长澄清流程阻塞
- 追问只发生在必要时，且仍留在搜索框内部
- 显式筛选器不被废弃，而是与 Agent 条件统一
- 搜索框外部页面保持熟悉感，不造成“整个产品变成聊天工具”的割裂

## 四、关键产品判断

### 4.1 新形态不是 AI 助手外挂，而是搜索框本体升级

本期的产品本体是：

`可原位扩展的搜索框容器`

而不是：

- 抽屉式 AI 助手
- 悬浮 chatbox
- 全页聊天工作台
- 独立“AI 搜索”页面

### 4.2 默认节奏：先搜后追问

自然语言输入后的默认流程固定为：

1. 系统先基于高置信条件执行一轮搜索
2. 用户先看到结果
3. 系统只针对高影响歧义发起轻量追问
4. 用户在搜索框内回答追问，结果继续 refine

不采用“先问清楚再搜索”的默认策略。

### 4.3 搜索框是唯一主控件

用户的以下动作都必须围绕同一个搜索框容器完成：

- 首次输入
- 追加一句新要求
- 回答系统追问
- 查看系统当前任务摘要
- 理解系统已提取的结构化条件

结果区只负责浏览结果，不承担主要对话职责。

### 4.4 搜索框外部保持原状

为了保持 Nox 搜索页的熟悉感与效率，本期默认不做这些区域的结构性重做：

- 页面导航
- 结果列表骨架
- 排序区
- 搜索框外的大部分筛选器位置

### 4.5 惊喜感来自局部 AI 化

这次的“AI 感”不靠全页改版，不靠聊天气泡流，而靠：

- 展开的搜索框拥有更独立的材质
- 搜索框内部组件有更强层次感
- 搜索框展开后像进入 `智能搜索模式`
- 但这种模式不扩散到外部页面

## 五、能力范围

### 5.1 本期包含

- 自然语言输入
- 关键词 / URL / 自然语言 / 混合输入共存
- 首轮结果召回
- 轻量追问 refine
- 结构化条件透明展示
- 用户手动改条件与 Agent 同步
- 零结果修正态
- unsupported / sensitive 受限响应
- 多平台共用模块，多页复用

### 5.2 本期明确不包含

- 跨平台混搜结果池
- 视频搜索 Agent 化
- 收藏、加名单、联系网红的 Agent 工作流
- 独立聊天产品式交互
- 搜索页全局视觉重做
- 底层索引与排序引擎重构

## 六、竞品结论压缩

### 6.1 直接可借鉴

- `Modash`
  - 自然语言直搜
  - 先结果后 refine
  - filters 继续收口
- `LinkedIn Recruiter`
  - AI recommendations 进入 current filters
  - AI 与 manual filters 在同一 search task
  - 结果解释增强信任
- `Klear`
  - AI 可作为可信能力层
  - 前台保持专业搜索工具感

### 6.2 应避免

- `HypeAuditor AI Scout` 式独立 scout/chat 作为默认入口
- `Upfluence Jaice` 式 campaign copilot 范围膨胀
- AI 与显式 filters 两套状态并行

### 6.3 Nox 的机会位

建立一个竞品尚未强占的形态：

- 追问留在搜索框内部
- 外部结果页保持稳定
- 搜索框展开态局部 AI 化

- 用户是否能直接描述需求而不是猜关键词
