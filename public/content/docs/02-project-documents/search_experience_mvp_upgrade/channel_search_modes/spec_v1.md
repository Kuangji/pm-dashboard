# 频道搜索模式升级 · Spec v1

## 一、信息架构

频道搜索页一级只分两个模式：

1. `标准搜索`
2. `AI搜索（智能精选）`

约束：

- `自然语言` 不作为一级模式存在
- `自然语言` 仅是 `标准搜索` 内部的显式输入形态

## 二、标准搜索

### 2.1 输入形态

标准搜索内提供：

- `关键词`
- `自然语言`

用户可显式切换。

### 2.2 关键词模式

- 保持现有关键词搜索体验
- 现有 schema 参数与结果页逻辑不改

### 2.3 自然语言模式

用户提交自然语言后：

1. 系统解析意图
2. 将结果回填到现有关键词 / 筛选参数体系
3. 触发现有标准搜索 API
4. 结果仍使用现有标准搜索结果页

约束：

- 不引入独立会话区
- 不引入独立 AI 结果协议
- 不显示解释层
- 用户感知重点是“更友好的建参方式”，不是“进入另一个产品”

## 三、AI搜索（智能精选）

### 3.1 起手方式

AI搜索支持两种起手：

1. `从 0 开始搜索`
2. `带入当前标准搜索条件开始 AI搜索`

约束：

- 允许桥接，不允许自动继承
- 用户必须主动触发带入当前条件
- 本轮实施主线以 `从 0 开始搜索` 为准
- `带入当前标准搜索条件开始 AI搜索` 仅保留入口与占位态，细节留到下一轮

### 3.2 输入与执行

- 输入形式：自然语言 query
- 每次搜索进入执行态
- 默认批次大小：`10`

### 3.3 结果消费

每个结果项提供四种状态：

- `unreviewed`
- `accepted`
- `excluded`
- `pending`

其中用户可主动设置：

- `accepted`
- `excluded`
- `pending`

### 3.4 批次推进

- 下一批只能由用户手动触发
- 不做自动阈值推进
- 下一批始终可点
- 用户直接进入下一批时，当前批次未处理项自动转为 `pending`

### 3.5 条件调整

- 用户可随时修改自然语言 query 重新发起搜索
- query 修改后，进入新的执行态
- query 修改后，仍留在当前 AI搜索 session

### 3.6 会话集合

AI搜索顶部展示三类摘要：

- `已采纳`
- `待定`
- `排除`

要求：

- 默认展示计数
- 支持展开查看明细
- 计数范围是整个 session 累计
- `accepted` 仅进入本次 session 候选池
- 不直接触发收藏/加人/营销计划写入

## 四、跨模式关系

### 4.1 标准搜索 -> AI搜索

- 可一键带入当前条件
- 带入只影响本次 AI搜索 的初始条件
- 该桥接动作必须在标准搜索结果页中可见，而不是只存在于空白 AI搜索 首屏
- 桥接动作应明确告诉用户：
  - 会带入哪些当前条件
  - 不会改写当前标准搜索状态
  - 会进入一个新的 AI搜索 session

### 4.2 AI搜索 -> 标准搜索

- 不自动回写
- 不自动修改标准搜索关键词或筛选参数
- 若标准搜索已有结果，切回时固定落到标准结果页
- 若标准搜索尚无结果，切回时落到标准空白输入态

### 4.3 Session 保留

- `标准搜索` 保留自己的条件与结果状态
- `AI搜索` 保留自己的 session 摘要与批次消费状态
- 互切时不自动互相写回

## 五、验收标准

- 用户能在标准搜索里显式切到自然语言输入
- 自然语言提交后能看见现有参数体系被回填
- AI搜索能清楚呈现执行中、批次结果、顶部摘要与下一批动作
- 标准搜索与 AI搜索 不互相污染状态
