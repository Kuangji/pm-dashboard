# 频道搜索模式升级 · Spec v1

## 一、信息架构

频道搜索页一级只分两个模式：

1. `标准搜索`
2. `AI搜索（智能精选）`

约束：

- `自然语言` 不作为一级模式存在
- `自然语言` 仅是 `标准搜索` 内部的显式输入形态

## 二、标准搜索

### 2.1 输入形态

标准搜索内提供：

- `关键词`
- `自然语言`

用户可显式切换。

### 2.2 关键词模式

- 保持现有关键词搜索体验
- 现有 schema 参数与结果页逻辑不改
- `关键词` 以胶囊样式留在搜索框内，表示经典搜索里的“成词输入”

### 2.3 自然语言模式

进入自然语言模式时：

- 输入框默认空白
- 当前 active schema 必须显式可见
- 原先关键词模式里留在搜索框内的 `关键词`，在自然语言模式下并入 `当前生效条件`
- 用户可输入一条完整需求，也可输入“再加上…”、“改成…”这类增量语句

用户提交自然语言后：

1. 系统解析意图
2. 解析时可接收当前 active schema 作为上下文
3. 将结果回填到现有关键词 / 筛选参数体系
4. 触发现有标准搜索 API
5. 结果仍使用现有标准搜索结果页

解析语义约束：

- 支持 `整套重建`
- 支持 `增量调整`
- 系统按用户措辞判断本次更偏重建还是增调
- 若判断为增调，则默认作用在 `当前 active schema` 上，而不是历史某条自然语言的上下文上

约束：

- 不引入独立会话区
- 不引入独立 AI 结果协议
- 不显示解释层
- 用户感知重点是“更友好的建参方式”，不是“进入另一个产品”

### 2.4 自然语言结果页

自然语言成功后，标准结果页顶部固定为：

1. `紧凑自然语言输入框`
2. `当前生效条件`
3. `折叠历史时间线`

要求：

- 输入框始终可用，可继续输入下一句自然语言
- 搜索动作永远基于当前生效的 schema 参数
- 页面不单独展示参数 diff 层
- `带当前条件去 AI搜索` 挂在 `当前生效条件` 区域
- 切到 `关键词` 时，直接带着当前生效条件进入传统可编辑搜索态
- 切到 `关键词` 时，自然语言历史直接清空

### 2.5 自然语言历史

- 只记录成功解析并真正触发过标准搜索的自然语言
- 每条历史默认展示：`原句 + 结果摘要`
- 点击历史项时：
  - 只把原句回填到当前输入框
  - 不自动立即搜索
  - 若再次提交，仍按当前 active schema 重新解释

### 2.6 自然语言失败态

- 失败反馈以内联提示形式贴在输入框下方
- 不做独立错误页
- 若当前已有标准搜索结果，则旧结果继续保留
- 不自动切关键词模式
- 不自动跳 AI搜索

## 三、AI搜索（智能精选）

### 3.1 起手方式

AI搜索支持两种起手：

1. `从 0 开始搜索`
2. `带入当前标准搜索条件开始 AI搜索`

约束：

- 允许桥接，不允许自动继承
- 用户必须主动触发带入当前条件
- `从 0 开始搜索` 时，总池 = 全量频道库
- `带入当前标准搜索条件开始 AI搜索` 时，总池 = 标准搜索先收缩出的候选范围
- `带入当前标准搜索条件开始 AI搜索` 不再是独立确认页，而是 AI 工作台的预装状态

### 3.2 输入与执行

- AI工作台顶部固定为：
  1. `当前候选总池`
  2. `当前 query`
  3. `session 集合摘要`
  4. `当前批次结果`
- 输入形式：自然语言 query
- query 只保留当前正在使用的一条
- 每次搜索进入执行态
- 默认批次大小：`10`
- query 的作用是在 `当前候选总池` 上继续做漏斗式筛选，而不是重新定义总池

### 3.3 Session 模型与持久化

- AI 搜索不再只有单一工作台状态，而是支持多个独立 session
- 每个 session 都必须至少保留：
  - `总池来源 / 总池范围`
  - `当前 query`
  - `已采纳 / 已排除`
  - `当前批次`
  - `进度状态`
- session 之间完全隔离，不共享采纳/排除、query 或批次进度
- session 需要在同一浏览器内持久化，刷新页面或离开再回来后仍可恢复
- 再次进入 AI 搜索时：
  - 若存在未归档 session，则默认恢复最近活动的那个
  - 若不存在未归档 session，则进入空白起手态

### 3.4 结果消费

- 当前批次结果卡只提供两个显式动作：
  - `accepted`
  - `excluded`
- `accepted / excluded` 单击即生效
- 生效后，该条结果立即从当前批次主列表中移除
- `pending` 不再作为正式动作或正式结果桶存在
- 未被采纳/排除的结果，只是当前仍留在主列表中的未处理项

### 3.5 批次推进

- 只要当前批次仍有未处理结果，下一批就只能由用户手动触发
- 若当前批次已被完整采纳/排除，则进入短过渡后自动开始下一批
- 用户在仍有未处理结果时主动进入下一批，这些未处理结果直接被放弃
- 当前批次结束后，结果区只保留新一批，不回堆旧批次长列表

### 3.6 条件调整

- 用户可随时修改自然语言 query 重新发起搜索
- query 修改后，进入新的执行态
- query 修改后，仍留在当前 AI搜索 session
- query 修改后，`当前候选总池` 不变
- query 修改后，`accepted / excluded` 集合继续保留

### 3.7 当前 session 集合

AI搜索顶部展示两类 session 集合：

- `已采纳`
- `排除`

要求：

- 默认展示计数
- 支持展开查看明细
- 支持从展开集合中轻量改判
- `accepted` 作为正向偏好信号持续影响后续批次
- `excluded` 在后续批次中作为硬过滤
- 不直接触发收藏/加人/营销计划写入
- `当前批次剩余未处理数量` 不放在顶部主摘要里，而放在结果区底部

### 3.8 Session 侧栏

- AI 搜索页改为 `左侧 session 历史侧栏 + 右侧当前工作台`
- session 历史侧栏应允许前端实时 `展开 / 收起`
- 展开 / 收起是同一个工作台里的 UI 状态，不引入独立 scene
- 左侧侧栏包含：
  - `新建 session`
  - `进行中 / 最近使用`
  - 折叠的 `已归档`
- `从 0 开始搜索` 每次明确新建一个 `global session`
- 从标准搜索点击 `带当前条件去 AI搜索` 时，每次都新建一个 `standard_search_scope session`
- 侧栏条目默认展示：
  - session 标题
  - 总池来源
  - 状态优先信息
  - 最近更新时间与已采纳/已排除计数
- session 支持：
  - 手动改名
  - 归档
  - 从归档区恢复并打开

## 四、跨模式关系

### 4.1 标准搜索 -> AI搜索

- 可一键带入当前条件
- 带入的核心是 `标准搜索当前候选范围`
- 该桥接动作必须在标准搜索结果页中可见，而不是只存在于空白 AI搜索 首屏
- 桥接动作应明确告诉用户：
  - 会形成怎样的 AI 搜索总池
  - 不会改写当前标准搜索状态
  - 会进入一个新的 AI搜索 session

### 4.2 AI搜索 -> 标准搜索

- 不自动回写
- 不自动修改标准搜索关键词或筛选参数
- 若标准搜索已有结果，切回时固定落到标准结果页
- 若标准搜索尚无结果，切回时落到标准空白输入态

### 4.3 Session 保留

- `标准搜索` 保留自己的条件与结果状态
- `AI搜索` 保留自己的 session 列表、active session、各 session 的总池 / query / 已采纳 / 已排除 / 批次状态
- 互切时不自动互相写回

## 五、验收标准

- 用户能在标准搜索里显式切到自然语言输入
- 自然语言提交后有轻处理中态，并能看见现有参数体系被回填
- 自然语言结果页顶部顺序稳定为：输入框 -> 当前生效条件 -> 折叠历史
- 自然语言结果页里继续输入新一句时，系统仍按当前 active schema 解释
- 切到关键词模式时，当前条件被直接迁移，且自然语言历史被清空
- 自然语言解析失败时，输入框下出现内联提示，旧结果仍保留
- `从 0 开始` 时，AI 搜索总池显示为全量频道库
- `带标准搜索条件起手` 时，AI 搜索总池显示为标准搜索范围
- AI 搜索左侧能看到 session 历史侧栏，并能高亮当前 active session
- 从标准搜索连续两次桥接到 AI 搜索时，会得到两个独立 session
- 在不同 session 间切换时，query、摘要与当前批次互不污染
- 刷新页面后，最近活动的未归档 session 能被恢复
- session 支持改名、归档与恢复
- AI 搜索顶部顺序稳定为：总池 -> query -> session 集合 -> 当前批次
- 当前批次卡片点击 `采纳 / 排除` 后，会立即从当前列表中移除
- 顶部主摘要只保留 `已采纳 / 已排除`
- `已采纳 / 已排除` 展开后可回看并轻量改判
- `改 query` 后，总池不变，session 不变，已采纳 / 已排除继续保留
- 当前批次有未处理结果时，下一批必须手动触发
- 当前批次被完整处理后，可自动进入下一批
- AI搜索能清楚呈现执行中、批次结果、顶部摘要与下一批动作
- 标准搜索与 AI搜索 不互相污染状态
