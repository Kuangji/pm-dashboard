# 频道搜索模式升级 · 旧参考拆解

## 参考来源

### 1. 营销计划页旧 `智能精选`

探索来源：

- 用户提供的线上页面与截图
- 实际浏览器交互
- 网络请求验证

关键事实：

- 页面没有独立 URL，只能在营销计划页中通过模式切换触发
- 相关请求命名空间为 `campaign2/*`
- `searchCondition` 编码参数可解出：

```json
{"searchType":1,"campaignId":"1103280896997347328"}
```

结论：

- 它是强绑定 `campaignId` 的计划内搜索能力
- 不是可直接搬到频道搜索页的全局搜索模式

### 2. `channel_search_agent_upgrade`

可复用背景：

- 搜索页外部尽量保持原状
- 搜索体验应主要在搜索区内部完成
- 自然语言、显式条件与结果联动要避免“外挂感”

不可直接复用部分：

- 完整状态机
- 追问流
- 条件 chips / 解释层
- URL / 混合输入的完整兼容路径

## 旧 `智能精选` 可借鉴点

- 独立于关键词搜索的模式心智
- 自然语言输入作为唯一主要输入
- 有“任务执行中”的过程感
- 结果卡与主结果页骨架可以复用

## 旧 `智能精选` 不可直接复用点

- 强绑定营销计划上下文
- 结果召回范围明显受当前计划影响
- 缺乏适合频道搜索的参数回落机制
- 缺乏清晰的“从 0 开始搜索 / 带当前条件细调”双起手设计
- 批次结果消费逻辑只有雏形，缺少稳定可用的交互细节

## 本次设计上的直接影响

### 标准搜索

- 不能简单用旧 `智能精选` 替代频道搜索
- 必须保持“最终回落为标准 schema 参数”的搜索心智

### AI搜索

- 可借鉴“自然语言 + 独立模式 + 过程感”
- 但需要重新定义为频道搜索里的探索式批次搜索工作流

## 本次冻结结论

- 频道搜索页一级结构：`标准搜索 | AI搜索`
- `自然语言` 是标准搜索内部的输入形态，不是一级模式
- `AI搜索` 不继承营销计划上下文
- `AI搜索` 允许从标准搜索一键带入当前条件，但不自动双向同步
