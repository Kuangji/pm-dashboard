# 频道搜索模式升级 · Design v1

## 一、整体设计原则

- 搜索页外部尽量稳定
- 模式切换尽量清晰
- `标准搜索` 看起来像原能力增强
- `AI搜索` 看起来像独立工作流，但不做成聊天产品

## 二、标准搜索设计

### 2.1 搜索区结构

在标准搜索搜索区内提供轻量形态切换：

- `关键词`
- `自然语言`

建议：

- 放在搜索框上沿或输入区左侧
- 视觉上显式但不喧宾夺主

### 2.2 关键词模式

- 尽量与现有搜索页一致
- 不主动引入 AI 面板感

### 2.3 自然语言模式

输入框表现为：

- 一条更长的自然语言输入区
- 明确的提交按钮
- 示例文案偏完整需求表达

提交后页面动作：

- 不是进入新结果系统
- 而是把用户拉回现有标准搜索心智

## 三、AI搜索（智能精选）设计

### 3.1 页面气质

AI搜索应该有独立模式感，但仍是搜索工作台，而不是聊天对话窗。

设计重心：

- 自然语言输入
- 批次结果消费
- 顶部摘要
- 下一批动作

### 3.2 空白起手态

应包含：

- query 输入框
- `开始 AI搜索`
- 次级按钮：`带入当前标准搜索条件`

### 3.3 批次执行中

需要同时呈现：

- 当前 query
- 正在执行的状态条或状态卡
- 结果区 skeleton

避免：

- 页面只剩空白
- 用户不确定是否还在运行

### 3.4 批次结果态

结果列表保持专业搜索工具样式：

- 左侧频道身份
- 中部指标
- 右侧最近内容或缩略信息

每张卡新增：

- `采纳`
- `排除`
- `待定`

### 3.5 顶部摘要

顶部摘要采用：

- 三段计数摘要
- 可展开明细

目标：

- 让用户知道本次 session 已经消费了哪些结果
- 但不把页面做成重型侧栏工作台

### 3.6 下一批动作

- 必须是稳定、显式的主按钮
- 放在当前批次消费区的自然结束位置
- 用户点击后再进入下一轮执行态

## 四、关键画面清单

- 标准搜索 / 关键词基线
- 标准搜索 / 自然语言输入
- 标准搜索 / 自然语言回填
- AI搜索 / 空白起手
- AI搜索 / 带条件起手
- AI搜索 / 执行中
- AI搜索 / 批次结果
- AI搜索 / 摘要展开
- AI搜索 / 修改 query 后重新搜索
