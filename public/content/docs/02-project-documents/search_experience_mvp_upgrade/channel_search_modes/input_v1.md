# 频道搜索模式升级 · Input v1

> 范围：需求 1 + 需求 2

## 一、背景

当前频道搜索相关的 AI 化能力处于分裂状态：

- 频道搜索页已经有 chatbox、智能扩词等辅助能力
- 营销计划页存在旧 `智能精选` 雏形
- `channel_search_agent_upgrade` 已经探索过完整 Agent 化方向

但这三者都不能直接作为本次 MVP 的可落地答案。

本次目标是先把搜索体验拆成两条真实能力线：

1. **标准搜索的自然语言化**
2. **独立 AI搜索（智能精选）**

## 二、问题定义

### 2.1 标准搜索的问题

现有标准搜索已经有稳定的 schema 参数与结果页，但用户仍要自己理解：

- 关键词怎么写
- 条件怎么配
- 筛选器如何组合

这导致自然语言需求无法自然进入现有标准搜索体系。

### 2.2 旧 `智能精选` 的问题

旧 `智能精选` 有雏形，但它：

- 强绑定营销计划上下文
- 不是频道搜索可直接复用的页面
- 批次消费与条件调优逻辑都不完整

## 三、目标

### 3.1 标准搜索目标

让用户在频道搜索中，可以显式切换到自然语言输入形态，用一句话构建与调整现有 schema 参数，并继续使用熟悉的标准结果页。

### 3.2 AI搜索目标

建立一个独立的探索式 `AI搜索（智能精选）` 模式：

- 用户用自然语言起搜
- 系统按批次返回少量结果
- 用户逐项消费结果
- 用户手动触发下一批
- 用户可随时改 query

## 四、用户路径

### 路径 A：标准搜索 / 关键词

- 用户使用原有关键词搜索
- 这是基线能力，不应回归

### 路径 B：标准搜索 / 自然语言

- 用户切换到自然语言输入
- 输入一句完整需求
- 系统解析并回填现有关键词/筛选参数
- 使用现有标准搜索结果页展示结果

### 路径 C：AI搜索（智能精选）

- 用户切到 AI搜索
- 选择从 0 开始，或一键带入当前标准搜索条件
- 输入自然语言
- 逐批消费结果

## 五、范围冻结

### 本期包含

- 标准搜索中的 `关键词 / 自然语言` 输入形态切换
- 自然语言到 schema 参数的可见回填
- 独立 `AI搜索（智能精选）`
- 批次结果消费：`采纳 / 排除 / 待定`
- `下一批` 手动触发
- `AI搜索` 顶部摘要与可展开明细

### 本期不包含

- 完整 Agent 搜索框状态机
- 会话追问
- 结果解释层
- URL / 混合输入的完整统一方案
- `AI搜索` 与收藏/营销计划的深度业务联动

## 六、成功标准

- 用户能明确区分 `标准搜索` 与 `AI搜索`
- 标准搜索的自然语言输入能成功回落到现有 schema 参数体系
- `AI搜索` 的批次消费逻辑可被直观看懂
- 用户能在 `AI搜索` 中持续消费与调优结果，而不是一次性只看首批
