# 相似网红替换 · Spec v1

## 一、入口结构

`相似网红` 保持独立入口，不并入：

- 标准搜索
- AI搜索（智能精选）

## 二、新默认流程

### 2.1 首屏输入

首屏默认是等待输入频道的空白态，并支持单频道种子输入。

输入来源：

- `search_select`
- `url`
- `handle`

输入模型约束：

- 默认是一个智能混合框，不拆成三个并列入口
- 搜索关键词必须先选中一个明确频道，才能形成有效种子
- URL / `@handle` 在显式提交时解析并锁定
- 主按钮只有在形成有效种子后才可用
- 无有效种子时，阻塞反馈显示在输入框下方

### 2.1.1 有效种子后的展示态

一旦形成有效种子：

- 顶部输入壳层不再继续保留可输入文本框
- 切换为频道卡展示态
- 明确显示频道头像、频道名、handle、平台
- 提供 `更换频道` 动作，用于回到可编辑混合框

### 2.2 URL 直达

支持通过 URL 携带种子频道信息进入相似网红页。

默认参数层级：

- `platform`
- `channelId`

行为：

- 进入页面后自动开始搜索
- 不停留在空白态
- 先进入执行中，再进入结果页

### 2.3 执行中

相似网红计算时间较长时，必须有独立执行中态。

要求：

- 清楚展示当前种子频道
- 结果页骨架或加载占位可见
- 输入区在执行中暂时锁定
- 已知种子直达时，顶部直接进入频道卡展示态

### 2.4 结果态

结果返回后，页面需要清楚展示：

- 当前种子频道
- 平台
- 结果列表
- 页顶仍保留当前种子输入壳层，默认是频道卡展示态
- 用户点击 `更换频道` 后，顶部回到可编辑混合框
- 旧结果在新种子正式提交前不立刻清空

## 三、结果形态

新默认召回保持普通结果列表形态：

- 标准结果列表
- 默认按相似程度排序
- 不做 AI搜索 那种批次消费流程

## 四、旧流程保留

页内保留次级 Tab：

- `收藏夹相似搜索`

约束：

- 不抢新默认流程主视觉
- 但必须保持可访问

## 五、快捷入口

以下入口可作为快捷起手方式：

- 频道搜索结果卡上的 `找相似`
- 频道详情页上的 `找相似`

落地规则：

- 统一落到相似网红独立页
- 已知种子频道时，通过 URL 带种子信息进入
- 先进入执行中，再进入结果列表

## 六、验收标准

- `相似网红` 默认首屏是单频道输入，不是收藏夹选择
- 搜索型输入在未选中频道前不能直接作为种子提交
- 有效种子后顶部能从输入态切换为频道卡展示态
- 旧收藏夹流程仍存在，但降级为次级 Tab
- 新结果形态是普通相似结果列表，不误入 AI搜索 批次流
