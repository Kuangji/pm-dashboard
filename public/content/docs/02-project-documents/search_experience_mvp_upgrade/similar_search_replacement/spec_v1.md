# 相似网红替换 · Spec v1

## 一、入口结构

`相似网红` 保持独立入口，不并入：

- 标准搜索
- AI搜索（智能精选）

## 二、新默认流程

### 2.1 首屏输入

首屏默认支持单频道种子输入。

输入来源：

- `search_select`
- `url`
- `handle`

### 2.2 种子锁定

种子频道锁定后，页面需要清楚展示：

- 频道身份
- 平台
- 基础摘要

## 三、结果形态

新默认召回保持普通结果列表形态：

- 标准结果列表
- 默认按相似程度排序
- 不做 AI搜索 那种批次消费流程

## 四、旧流程保留

页内保留次级 Tab：

- `收藏夹相似搜索`

约束：

- 不抢新默认流程主视觉
- 但必须保持可访问

## 五、快捷入口

以下入口可作为快捷起手方式：

- 频道搜索结果卡上的 `找相似`
- 频道详情页上的 `找相似`

落地规则：

- 统一落到新默认相似网红流程
- 已知种子频道时，直接进入“种子已锁定 + 相似结果列表”

## 六、验收标准

- `相似网红` 默认首屏是单频道输入，不是收藏夹选择
- 旧收藏夹流程仍存在，但降级为次级 Tab
- 新结果形态是普通相似结果列表，不误入 AI搜索 批次流
