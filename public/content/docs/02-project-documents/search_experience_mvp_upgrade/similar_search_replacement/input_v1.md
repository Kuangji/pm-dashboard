# 相似网红替换 · Input v1

> 范围：需求 3

## 一、背景

现有 `相似网红搜索` 默认依赖收藏夹输入，且召回逻辑基于标签聚合与重合度计算。

本次迭代要完成的是：

- 用 `单个网红频道` 作为主输入
- 用 `多模态向量运算` 作为主召回逻辑
- 用新流程替代旧默认入口
- 旧流程保留为次级入口

## 二、目标

### 2.1 主目标

把 `相似网红` 默认入口改成更轻、更直接的“以一个频道为种子找相似频道”。

### 2.2 次目标

保留旧收藏夹流程，避免老用户完全断路。

## 三、用户路径

### 路径 A：新默认流程

- 进入 `相似网红`
- 搜索 / 粘贴 / 输入一个频道
- 锁定种子频道
- 查看相似结果列表

### 路径 B：旧次级流程

- 进入 `相似网红`
- 手动切换到 `收藏夹相似搜索`
- 使用旧的收藏夹输入逻辑

## 四、范围冻结

### 本期包含

- 新默认首屏
- 单频道输入
- 多模态向量召回结果列表
- 旧流程次级 Tab

### 本期不包含

- 将相似网红并入 AI搜索
- 将相似网红改成批次消费式工作流
- 底层向量算法细节对外解释

## 五、成功标准

- 新用户不依赖收藏夹，也能直接开始相似搜索
- 老流程仍然可达
- 用户能一眼看出新旧主次关系
