# 相似网红替换 · 当前流程拆解

## 现有默认流程（用户描述）

旧 `相似网红搜索` 的核心逻辑是：

1. 用户选择一个 `收藏夹` 作为输入
2. 系统读取收藏夹内所有网红
3. 基于标签聚合与重合度计算召回相似网红

## 旧流程的优点

- 适合已有积累用户
- 用“收藏夹”作为输入，样本稳定
- 能自然利用历史人工筛选结果

## 旧流程的问题

### 1. 默认入口过重

用户必须先有收藏夹，才能开始相似搜索。

### 2. 输入粒度不对

当用户只是想围绕“一个明确频道”找相似对象时，收藏夹输入显得过重。

### 3. 计算语义偏旧

旧方案核心依赖：

- 标签聚合
- 重合度计算

这不再是本次想主推的相似召回逻辑。

## 新替换方向

默认入口改为：

`单频道输入 -> 多模态向量召回`

旧流程保留，但降为页内次级入口。

## 冻结结论

- `相似网红` 继续保留独立入口
- 新默认主流程不并入频道搜索或 AI搜索
- 旧收藏夹流程仍保留可访问性，但不占默认主入口
