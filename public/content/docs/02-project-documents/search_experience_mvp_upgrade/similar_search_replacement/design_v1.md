# 相似网红替换 · Design v1

## 一、整体设计目标

新相似网红页应该让用户感知到：

- 默认入口更轻
- 起手更明确
- 结果仍然专业且可信

不应该让用户感知到：

- 被迫先维护收藏夹
- 被拉进另一套复杂 AI 工作流

## 二、首屏设计

### 2.1 主输入区

首屏主输入区支持：

- 搜索频道
- 粘贴 URL
- 输入 handle

建议视觉：

- 大输入框
- 清楚的“种子频道”心智提示
- 一个明显的主按钮：`开始找相似网红`

### 2.2 旧流程入口

使用页内次级 Tab：

- 主 Tab：`频道种子搜索`
- 次级 Tab：`收藏夹相似搜索`

要求：

- 次级 Tab 可见，但不抢默认入口

## 三、种子锁定态

当频道被锁定后，应出现一个稳定的种子卡：

- 频道头像 / 名称 / handle
- 平台
- 简短说明：`已围绕该频道召回相似网红`

## 四、结果列表

结果区沿用专业搜索结果卡风格：

- 频道身份
- 核心指标
- 相似相关的排序感

无需新增：

- 批次消费
- 会话总结
- 采纳 / 排除 / 待定

## 五、关键画面清单

- 新默认首屏
- 种子输入中
- 种子锁定 + 相似结果列表
- 旧收藏夹流程次级 Tab
