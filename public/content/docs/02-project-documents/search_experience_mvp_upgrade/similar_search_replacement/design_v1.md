# 相似网红替换 · Design v1

## 一、整体设计目标

新相似网红页应该让用户感知到：

- 默认入口更轻
- 起手更明确
- 结果仍然专业且可信

不应该让用户感知到：

- 被迫先维护收藏夹
- 被拉进另一套复杂 AI 工作流

## 二、首屏设计

### 2.1 主输入区

首屏主输入区支持：

- 搜索频道
- 粘贴 URL
- 输入 handle

建议视觉：

- 大输入框
- 清楚的“种子频道”心智提示
- 一个明显的主按钮：`开始找相似网红`
- 辅助提示：也支持带 `平台 + 频道ID` 的 URL 直达

交互要求：

- 默认是一个智能混合框，不拆三列入口
- 搜索关键词必须从候选结果中选中频道后，才算形成有效种子
- URL / `@handle` 在显式提交时解析
- 无有效种子时，主按钮不可用，且输入框下方显示内联阻塞提示

### 2.1.1 有效种子后的输入壳层

一旦形成有效种子，输入壳层应从“混合框编辑态”切换为“频道卡展示态”：

- 显示头像、频道名、handle、平台
- 明确标识：这是“当前频道种子”
- 不再保留继续输入的文本框
- 通过明确动作 `更换频道` 回到编辑态

### 2.2 旧流程入口

使用页内次级 Tab：

- 主 Tab：`频道种子搜索`
- 次级 Tab：`收藏夹相似搜索`

要求：

- 次级 Tab 可见，但不抢默认入口

## 三、执行中态

当频道已确定并开始计算后，应出现独立执行中态：

- 页顶显示当前种子频道
- 明确状态：`正在围绕该频道搜索相似网红`
- 结果区用骨架承接
- 页顶输入区暂时锁定，避免用户误解系统正在算哪个频道

## 四、结果态

结果返回后：

- 页顶继续保留同一个输入壳层，默认是频道卡展示态
- 用户可通过 `更换频道` 进入编辑态
- 编辑新种子时旧结果先保留，直到新提交后才进入新的执行中
- 当前种子信息仍然清楚可见

## 五、结果列表

结果区沿用专业搜索结果卡风格：

- 频道身份
- 核心指标
- 相似相关的排序感

无需新增：

- 批次消费
- 会话总结
- 采纳 / 排除 / 待定

## 六、关键画面清单

- 新默认首屏
- 执行中
- 相似结果列表
- 旧收藏夹流程次级 Tab
