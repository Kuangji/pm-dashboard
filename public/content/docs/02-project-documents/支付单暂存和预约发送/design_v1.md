## 一、功能概述

本次上线两个功能：**支出单暂存（草稿）** + **预约发送**。两者共享同一个创建支出单弹窗，通过弹窗底部操作区进行统一交互入口。

---

## 二、功能一：支出单暂存（草稿）

### 2.1 创建表单底部操作区

创建支出单弹窗底部操作区，三个操作按主次分层排列：

```
┌─────────────────────────────────────────────────────────────────┐
│  [保存草稿]（线框次要按钮，左侧）      [立即发送]  [预约发送]（右侧）  │
└─────────────────────────────────────────────────────────────────┘
```

| 操作 | 按钮样式 | 位置 |
|------|---------|------|
| 保存草稿 | 线框次要按钮（灰色边框） | 左侧 |
| 立即发送 | 填充主要按钮（主色橙） | 右侧靠内 |
| 预约发送 | 线框主要按钮（主色橙边框） | 最右侧 |

**说明**：「立即发送」为默认主操作，视觉权重最高；「预约发送」为次要主操作；「保存草稿」为辅助操作，视觉权重最低。

---

### 2.2 「保存草稿」按钮状态

| 状态 | 触发条件 | 视觉表现 |
|------|---------|---------|
| 置灰（不可点） | 未选择网红频道 | 按钮置灰，光标 `not-allowed` |
| 正常可点 | 已选择网红频道 | 正常线框样式 |
| 加载中 | 点击后等待接口返回 | 按钮内显示 loading 转圈，文字变为「保存中…」，不可重复点击 |

**置灰时 Tooltip**：鼠标悬停于置灰按钮时显示 tooltip：

> 请先选择网红频道

---

### 2.3 关闭弹窗时的确认交互

#### 场景一：全新创建时关闭（二选）

**触发**：用户点击弹窗右上角「×」或底部「取消」按钮，且当前是全新创建（非继续编辑草稿）。

| 子场景 | 行为 |
|--------|------|
| 已选择网红频道 | 弹出「是否保存草稿？」确认弹窗 |
| 未选择网红频道 | 直接关闭弹窗，无任何提示 |

**「是否保存草稿？」确认弹窗**：

```
┌────────────────────────────────────┐
│         是否保存草稿？              │
│                                    │
│  您可以稍后在列表中继续编辑此草稿。  │
│                                    │
│  [不保存，直接关闭]   [保存草稿]    │
└────────────────────────────────────┘
```

| 按钮 | 操作 |
|------|------|
| 保存草稿 | 保存当前填写内容，关闭弹窗，列表顶部出现新草稿条目 |
| 不保存，直接关闭 | 不保存，直接关闭弹窗 |

---

#### 场景二：继续编辑已有草稿后关闭（三选）

**触发**：用户点击「×」或「取消」，当前是从草稿「继续编辑」打开的弹窗。

| 子场景 | 行为 |
|--------|------|
| 有修改未保存 | 弹出「尚未保存修改」确认弹窗 |
| 无修改（内容与上次保存完全一致） | 直接关闭弹窗，无提示 |

**「尚未保存修改」确认弹窗**：

```
┌──────────────────────────────────────────────┐
│            尚未保存修改                        │
│                                              │
│  您的修改尚未保存，是否保存？                  │
│                                              │
│  [取消]   [不保存，恢复到上次保存]   [保存草稿]  │
└──────────────────────────────────────────────┘
```

| 按钮 | 操作 |
|------|------|
| 保存草稿 | 覆盖保存，关闭弹窗 |
| 不保存，恢复到上次保存 | 放弃本次修改，关闭弹窗，草稿恢复至上次保存状态 |
| 取消 | 关闭此确认弹窗，**留在创建弹窗内继续编辑**（不执行关闭） |

---

### 2.4 草稿状态标签

**标签文本**：草稿

**样式规范**：
- 颜色：中灰色背景（置灰色系），深灰色文字
- 无图标前缀（与「进行中」「已取消」保持同等级别标签样式）
- 尺寸与现有状态标签一致

> 注：列表中现有「进行中」为蓝色，「已取消」为橙色；草稿为灰色，语义上为「未激活/未正式提交」。

---

### 2.5 草稿列表行

草稿条目**置顶展示**，位于所有正式支付单之前。

#### 字段展示

| 字段 | 草稿行的值 |
|------|-----------|
| 网红头像 + 频道名 | 已填，必有 |
| 支出单ID | **不显示**（草稿尚未生成正式 ID） |
| 创建时间 | 显示草稿的「最后编辑时间」（见下方说明） |
| 币种 | 已填则显示，未填则「—」 |
| 收款人将收到金额 | 已填则显示，未填则「—」 |
| 您共支出金额 | 已填则显示，未填则「—」 |
| 任务名称 | 已填则显示，未填则「—」 |
| 状态标签 | 「草稿」（灰色） |
| 进度时间线 | **不显示** |
| 操作按钮 | 「继续编辑」/「删除」 |

**最后编辑时间**（建议3实现）：

- 展示格式：相对时间 + 绝对时间，如「2 小时前编辑」，悬停 tooltip 显示完整时间戳（如 `2026-03-03 14:30`）
- 位置：与正式支付单「支出单ID + 创建时间」的位置对齐，复用同一显示区域

#### 操作按钮

| 操作 | 样式 | 行为 |
|------|------|------|
| 继续编辑 | 图标按钮（✏ 编辑图标），与正式支付单操作按钮风格一致 | 打开创建弹窗，恢复所有已保存字段内容 |
| 删除 | 图标按钮（🗑 删除图标） | 弹出删除确认弹窗 |

---

### 2.6 删除草稿确认弹窗

```
┌──────────────────────────────────┐
│         确认删除草稿？             │
│                                  │
│  删除后无法恢复，确认删除？        │
│                                  │
│      [取消]        [确认删除]     │
└──────────────────────────────────┘
```

| 按钮 | 操作 |
|------|------|
| 确认删除 | 删除图标主色红（危险操作）；点击后草稿记录永久删除，行从列表消失 |
| 取消 | 关闭弹窗，草稿保留 |

## 三、功能二：预约发送

### 3.1 「预约发送」展开时间选择器

**触发**：用户点击弹窗底部「预约发送」按钮。

**交互流程**：

```
用户点击「预约发送」
  → 按钮变为选中态（橙色边框高亮）
  → 在「保存草稿 / 立即发送 / 预约发送」行下方展开时间选择区域
  → 显示日期 + 小时选择器
  → 用户完成选择后，再次点击「预约发送」（此时改为「确认预约」）正式提交
```

**时间选择区域布局**：

```
┌──────────────────────────────────────────────────────────┐
│  发送时间                                                 │
│                                                          │
│  快速选择：[+3天]  [+7天]  [+14天]                        │
│                                                          │
│  [ 请选择日期         ⌵ ]   [ 小时: 00 ▾ ]               │
│  （点击展开日历面板）                                       │
│                                                          │
│  ⏰ 请在预约时间的 24 小时前完成网红签合同，否则审核可能       │
│     无法完成，导致预约失败。                                 │
│                                                          │
│  时区：UTC+8（北京时间）                                   │
└──────────────────────────────────────────────────────────┘
```

#### 快速选择（建议1实现）

| 快选选项 | 行为 |
|---------|------|
| +3天 | 自动填入「当前时间 + 72 小时」对应的日期，小时取当前小时 |
| +7天 | 自动填入「当前时间 + 7 天」对应的日期，小时取当前小时 |
| +14天 | 自动填入「当前时间 + 14 天」对应的日期，小时取当前小时 |

> 快选后用户仍可手动调整日期/小时。

#### 日历面板（日期选择）

- 点击日期输入框后弹出日历面板
- **3天内日期**（从当前时刻起 72 小时内的日期）：置灰不可选
- 日历默认展示当前月；默认选中最早可选日期（当前日期 + 3天）
- 用户可跨月浏览，无最晚日期限制（UI 建议默认展示未来 3 个月内）

#### 小时下拉

- 选项：00、01、02 … 23（24 小时制）
- 若所选日期为最早可选日（恰好 +3 天），则不足 72 小时的小时数置灰不可选
  - 示例：当前时间 14:30，最早可选日期为 3 天后，则该日期下 00~13 小时置灰

#### 时区标注

- 固定展示于时间选择器下方：「时区：UTC+8（北京时间）」
- 如未来平台需支持多时区，此处保留扩展空间

#### 24h 提示文案

固定展示于时间选择器区域内（非弹窗），始终可见：

> ⏰ 请在预约时间的 24 小时前完成网红签合同，否则审核可能无法完成，导致预约失败。

---

### 3.2 列表展示 — B-b 预约等待阶段

#### 时间线节点

支付单处于 B-b（预约等待中）时，进度时间线第三节点：

```
●（橙色实心）支出单已创建 — 2026-03-03 14:00
●（橙色实心）平台审中 — 2026-03-04 10:00
○（橙色空心）开始转账（预约中）— 预约于 2026-03-10 14:00
```

- 节点文字：「开始转账（预约中）」
- 节点下方显示预约时间戳：如 `预约于 2026-03-10 14:00（UTC+8）`

#### 倒计时气泡（建议2实现）

- 节点**不内联显示倒计时**（避免占位）
- 在「开始转账（预约中）」节点旁放置一个信息图标（ℹ）
- 鼠标悬停于图标时，弹出气泡（Popover/Tooltip）显示：

  > 还有 **3 天 12 小时** 后自动发送

- 气泡为只读信息，无操作按钮

#### 操作按钮

B-b 阶段列表行操作按钮区：

```
👁查看 | 修改预约时间 | 转为立即发送 | 取消
```

| 操作 | 点击行为 |
|------|---------|
| 修改预约时间 | 打开「修改预约时间」弹窗（见 §3.3） |
| 转为立即发送 | 弹出二次确认弹窗（见 §3.4） |
| 取消 | 复用现有「取消支付单」确认弹窗 |

---

### 3.3 修改预约时间弹窗

**触发**：状态 A 或 B-b 下，点击「修改预约时间」。

```
┌──────────────────────────────────────────────┐
│              修改预约时间                      │
│                                              │
│  当前预约时间：2026-03-10 14:00（UTC+8）      │
│                                              │
│  新预约时间：                                 │
│  快速选择：[+3天]  [+7天]  [+14天]            │
│  [ 请选择日期         ⌵ ]   [ 小时: 00 ▾ ]   │
│                                              │
│  ⏰ 请确认合同已安排，避免预约失败。            │
│                                              │
│         [取消]         [确认修改]             │
```

- 时间选择交互与创建时一致（日历 + 小时下拉 + 快选）
- 最早可选时间：
  - 状态 A：当前操作时刻 + 72h
  - 状态 B-b：当前操作时刻 + 72h

### 3.4 「转为立即发送」二次确认弹窗

**触发**：状态 A/A1/B-b 下，点击「转为立即发送」。

```
┌──────────────────────────────────────────────┐
│           确认转为立即发送？                   │
│                                              │
│  转换后，系统将在审核通过后立即触发转账，       │
│  当前预约时间设置将失效。                      │
│                                              │
│          [取消]       [确认]                  │
└──────────────────────────────────────────────┘
```

- 确认后：取消预约，支付单按立即发送逻辑继续流转
- 操作按钮区：状态 A/A1 下的「转为立即发送」移除，状态恢复正常列表操作

---

### 3.5 状态 A/A1 列表操作按钮（预约发送专有）

**适用范围**：该支付单为预约发送 + 状态为 A 或 A1。

**状态 A 操作区**：

```
👁查看 | ✏编辑 | 修改预约时间 | 转为立即发送 | 取消
```

**状态 A1 操作区**：

```
👁查看 | 转为立即发送 | 取消
```

> 状态 A1（审核中）不可修改预约时间（规则见 spec §3.5），因此不显示「修改预约时间」。

---

### 3.6 A4/A5 预约超时操作入口

**触发**：预约超时（A4/A5 状态），广告主在站内信中查看通知后，可通过通知内的行动入口操作；或回到支付单列表操作。

#### 列表行操作（预约超时状态 A4/A5）

```
👁查看 | 重新预约 | 转为立即发送 | 作废
```

| 操作 | 点击行为 |
|------|---------|
| 重新预约 | 复用「修改预约时间」弹窗（与 §3.3 相同），最早可选时间 = 当前时刻 + 72h；确认后时间线即时更新 |
| 转为立即发送 | 弹出「确认转为立即发送？」确认弹窗 |
| 作废 | 弹出「确认作废？」确认弹窗（不可恢复） |

> **操作失败路径**（等接口策略）：以上三个操作若接口返回失败，按钮恢复可点状态，Toast 提示「操作失败，请重试」（红色，3秒）。前端不区分失败原因，统一提示重试。

#### 「确认作废？」弹窗

```
┌──────────────────────────────────────────────┐
│              确认作废此支付单？                 │
│                                              │
│  作废后支付单无法恢复，余额将退还至账户余额。    │
│                                              │
│          [取消]       [确认作废]               │
└──────────────────────────────────────────────┘
```

## 四、组件状态清单

### 创建支出单弹窗

| 组件 | 条件 | 状态 |
|------|------|------|
| 「保存草稿」按钮 | 未选网红频道 | 置灰，不可点，悬停显示 tooltip |
| 「保存草稿」按钮 | 已选网红频道 | 可点击 |
| 「保存草稿」按钮 | 点击后接口返回中 | loading 状态，文字「保存中…」，不可重复点击 |
| 「立即发送」按钮 | 通用 | 始终可点（提交时校验必填项） |

| 时间选择区域 | 已点击「预约发送」 | 展开显示 |
| 日历面板 — 日期 | 3天内（< 72h） | 置灰不可选 |
| 小时下拉 — 选项 | 同为最早可选日且小时不足 72h | 置灰不可选 |
| 快选按钮（+3/7/14天） | 通用 | 始终可点 |

| 组件 | 条件 | 状态 |

## 五、操作反馈规格

| 操作 | 操作完成后 UI 响应 |
|------|-----------------|
| 保存草稿（成功） | 弹窗关闭；列表顶部出现草稿条目；Toast 成功提示「草稿保存成功」（绿色，3秒） |
| 保存草稿（失败） | 弹窗保持开启；Toast 错误提示「保存失败，请重试」（红色，3秒） |
| 删除草稿（确认） | 弹窗关闭；草稿行从列表消失；Toast 提示「草稿已删除」（中性，3秒） |
| 立即发送（成功） | 弹窗关闭；草稿条目（若有）自动消失；列表出现新正式支付单（状态 A） |
| 预约发送（成功） | 弹窗关闭；草稿条目（若有）自动消失；列表出现新正式支付单（状态 A，进度线第三节点标注「预约中」） |
| 修改预约时间（成功） | 弹窗关闭；列表行时间线更新为新预约时间；Toast 提示「预约时间已更新」（绿色，3秒） |
| 转为立即发送（确认） | 弹窗关闭；列表行操作按钮恢复为正常（去掉「修改预约时间」/「转为立即发送」）；Toast 提示「已转为立即发送」（绿色，3秒） |
| 作废支付单（确认） | 弹窗关闭；列表行状态更新为「已取消」 |
| 系统自动触发 B-b→B-c | 无前端实时更新（用户刷新页面后可见时间线更新）；预约时间节点显示已激活 |
| A4/A5「重新预约」成功 | 弹窗关闭；列表行时间线即时更新为新预约时间；Toast 提示「预约时间已更新」（绿色，3秒）；支付单状态恢复预约等待中 |
| A4/A5「重新预约」失败 | 弹窗保持开启；按钮恢复可点；Toast 提示「操作失败，请重试」（红色，3秒） |
| A4/A5「转为立即发送」成功 | 弹窗关闭；列表行操作按钮恢复正常；Toast 提示「已转为立即发送」（绿色，3秒） |
| A4/A5「转为立即发送」失败 | 弹窗关闭；按钮恢复可点；Toast 提示「操作失败，请重试」（红色，3秒） |
| A4/A5「作废」成功 | 弹窗关闭；列表行状态更新为「已取消」 |
| A4/A5「作废」失败 | 弹窗关闭；按钮恢复可点；Toast 提示「操作失败，请重试」（红色，3秒） |
