# 内容监控 2.0 (v7.6.0) 公共业务上下文 (Common Context)

## 1. 模块综述 (Module Overview)
- **模块名称**: 内容监控 (Content Monitor)
- **核心价值**: 通过对公开数据的自动化获取，为品牌提供多平台内容的实时数据追踪与分析。

## 2. 名词定义 (Definitions)
- **监控项目（Monitor Project）**：用户创建的任务集合。
- **监控任务 (Monitor Task)**: 单个已发布内容的抓取与分析。
- **自动追踪 (Auto-track)**: 预发布内容的规则化监听。
- **暂存区 (Staging Area)**: 创建前的纯前端缓冲区。
- **来源 (Source)**: 标识任务产生方式（手动录入/自动追踪）。

## 3. 基础 UI 结构 (Base UI Structure)
监控项目页面各模块应遵循以下 **从上至下** 的垂直布局顺序：
1.  **监控项目标题栏**: 页面最顶部，包含项目名称、导出等。
2.  **筛选器组 (Filter Group)**:
    *   **第一行 (通用筛选)**: 包含搜索框（搜频道/标题/链接）、下拉筛选列表（监控状态、地区、视频标签、负责人、监控时长、播放量、建立日期）。
    *   **第二行 (平台切换 TAB)**: 位于通用筛选下方。包含 全部、YouTube、TikTok、Instagram、Facebook、Twitter。每个 TAB 必须以 `名称 (数量)` 格式显示任务数，例如：`YouTube (45)`。
3.  **数据汇总面板**: 位于平台 TAB 下方，展示当前过滤后的统计卡片。
4.  **监控任务列表**: 位于最下方，展示具体的任务条目或卡片。

## 4. 视觉设计规范 (Visual Tokens for Stitch)
为确保不同批次生成的原型一致，所有 Stitch 指令必须包含：
- **圆角 (Border Radius)**: 弹窗/容器 `12px`, 按钮/输入框 `8px`, 卡片 `10px`。
- **间距 (Spacing)**: 页面边距 `24px`, 组件间距 `16px`, 列表行高 `48px (Compact)`。
- **按钮尺寸**: 标准高度 `36px`, 字体 `14px`, 权重 `Medium`。
- **色彩规范**: 
    - 主色: `NoxBlue (#2E5BFF)`
    - 背景: `LightGray (#F5F7FA)`
    - 文本: `PrimaryBlack (#1D1D1F)`, `SecondaryGray (#86868B)`
- **密度 (Density)**: `Compact` (紧凑型，适合 SaaS 后台)。
