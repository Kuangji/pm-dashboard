# 内容监控：新建监控模块重构需求文档 (7.6.0 v1)

## 1. 项目背景与现状分析
### 1.1 项目背景 (Project Background)
NoxInfluencer 旨在为品牌方和分析师提供全流程的网红营销解决方案。本项目作为内容监控 2.0 (v7.6.0) 的核心重构部分，旨在解决创建流程割裂、批量录入效率低以及自动追踪缺乏管理的问题。

### 1.2 业务现状 (Status Quo)
在旧版本中，内容监控的创建流程分为两个独立的入口：
1.  **“创建监控”按钮**：仅针对已发布的单个内容链接，采用简单的弹窗表单。
2.  **“自动追踪”按钮**：针对预发布内容，虽然有独立表单，但**缺乏管理界面**，用户无法查看、编辑或停止已建立的追踪规则，规则运行状态处于“黑盒”模式。

### 1.3 核心痛点 (Pain Points)
*   **入口割裂，认知成本高**：用户在“我有一个链接”和“我想监控某个频道”之间需要做决策选择不同的按钮，且两个功能的交互逻辑不统一。
*   **批量录入效率低下**：缺乏“暂存区”机制，用户在批量添加链接时无法进行统一的配置修改，报错反馈不直观（通常是报错后整个表单重置）。
*   **自动追踪无法闭环**：由于缺少规则管理列表，用户无法获知规则的转化效率（即一个规则产生了多少监控任务），且难以维护旧规则。
*   **数据链路断裂**：在主监控列表中，无法分辨哪些任务是手动创建的，哪些是自动生成的，导致后续数据分析困难。

### 1.2 核心目标
*   **统一体验**：消除“手动监控”与“自动追踪”的入口割裂。
*   **提升效率**：通过“暂存区”实现百级任务的快速录入、校验与批量配置。
*   **管理闭环**：在不跳转页面的前提下，完成自动追踪规则的“增删改查”。

## 2. 交互域：统一入口与布局
### 2.1 模态框定义
*   **触发点**：点击主页面的“新建监控”按钮。
*   **容器**：采用宽幅模态框（Large Modal, 建议宽度 1000px+）。
*   **导航**：顶部 Tab 分为：
    1.  **监控已发布内容**（默认激活）：针对已有 URL 的单条或批量监控。
    2.  **自动追踪预发布内容**：针对频道关键词的长期监听。

---

## 3. 功能域：监控已发布内容 (Tab 1)
### 3.1 录入暂存区 (Staging Area)
不再区分单条和批量，统一为“列表编辑”模式。
*   **输入源**：
    *   **快速粘贴**：支持文本框输入多个 URL（换行分隔）。
    *   **文件导入**：支持上传 CSV/Excel 模板。
    *   **追加逻辑**：新录入的数据通过“追加”方式进入暂存表格，不覆盖已有内容。
*   **即时解析**：输入 URL 后，前端触发后端 `/preview` 接口，在表格中实时显示视频封面、标题及所属平台。

### 3.2 配置继承与锁定逻辑
*   **通用配置区**：位于表格上方，可设置“监控时长”、“关联计划”、“标签”。
*   **继承机制**：
    *   默认状态下，通用配置的变更实时同步至表格内所有行。
    *   **锁定保护**：一旦用户手动修改了某行的特定参数（如 A 任务单独设置了 180 天），该字段进入“锁定状态”，不再响应通用配置变更。
    *   **恢复同步**：锁定字段旁提供“重置”按钮，点击后恢复跟随通用配置。

### 3.3 提交与容错机制
*   **部分提交策略**：
    *   点击“开始监控”，前端过滤出校验通过的任务发送至后端。
    *   **成功处理**：创建成功的任务从暂存区移除。
    *   **失败处理**：失败任务（链接失效、额度不足、格式错误）保留在暂存区，并高亮显示错误原因。
*   **进度反馈**：提交后显示全局 Loading 条，并展示进度（例如：“正在创建：15/50...”）。

## 4. 功能域：自动追踪管理 (Tab 2)
### 4.1 嵌入式管理列表
*   **布局**：左侧/顶部为“创建规则表单”，右侧/底部为“已有规则列表”。
*   **核心功能**：支持在 Tab 内直接进行：
    *   **开关切换**：一键开启/关闭某条追踪规则。
    *   **快捷编辑**：点击规则行，表单载入旧数据进入编辑模式。
    *   **统计透出**：显示该规则已自动触发创建了多少个监控任务。

### 4.2 配置增强
*   **频道导入**：搜索框除支持频道 ID/URL 外，新增“从收藏夹导入”按钮。
*   **冲突检测**：保存规则时，系统需校验 `频道 + 关键词` 的唯一性，防止重复扣费或逻辑冗余。

---

## 5. 数据域：溯源与列表增强
### 5.1 监控任务列表字段新增
*   **来源 (Source)**：
    *   枚举值：`手动创建` / `自动追踪`。
*   **溯源信息**：
    *   若为“自动追踪”，Hover/展示关联的规则名称及 ID，支持点击跳转至该规则的配置界面。

---

## 6. 非功能需求 (Optional)
*   **并发处理**：批量创建接口需支持至少 50 条/次的并发请求处理。
*   **暂存区持久化**：用户意外关闭模态框时，本地 Session 需保存暂存区数据，再次打开时提示“是否恢复上次未完成的内容”。
