# 原始输入：监控列表增强功能 (Module 2)

> **公共上下文引用**: 参见 `monitor_common_context_v1.md`

## 1. 核心业务诉求
针对自动追踪产生的任务进行“闭环”反馈，提升用户在列表页对任务来源、执行进度及新任务生成的感知度。

## 2. 初始灵感/功能点

### 2.1 来源溯源 (Source Traceability)
*   **字段**: 卡片需增加“来源”标签。
*   **标识**: 
    *   手动创建显示图标 `👤`。
    *   自动追踪显示图标 `🤖` 并标注规则名，支持点击跳转回管理界面。

### 2.2 新任务强提醒 (New Task Alert)
*   **构思**: 自动生成的任务，在首次被查看前，显示“未读”蓝点标记。

### 2.3 监控进度可视化 (Progress Visualization)
*   **构思**: 增加微型进度条，显示“已监控 N/M 天”，即将到期时高亮预警。

### 2.4 筛选增强
*   **构思**: 增加按“来源类型”筛选（全部、手动、自动）。

### 2.5 监控任务续期
*   **构思**：支持对监控任务进行续期操作。
    *   **操作方式**: 支持在列表行操作菜单中进行单条续期，以及勾选多个任务后进行批量续期。
    *   续期操作应提供多种时长选择（可选时长同新建监控任务保持一致）。
    *   续期后，任务的到期时间应在原到期时间基础上延长。
    *   续期成功后，任务状态和到期时间应实时更新。

### 2.6 关键变动异常告警 (Change Anomaly Alert)
*   **构思**：在列表数据列（如观看量、点赞量）中，通过视觉手段标识数据的异常波动。
    *   **爆发式增长**：识别近 24 小时内的爆发性数据增长，标注“爆款”标识。
    *   **数据停滞/异常**：对于长时间无数据更新或数据停滞的任务，通过红字或图标预警，提醒用户检查（如视频是否被转为私密）。
    *   **异常原因透出**：Hover 预警图标时，显示可能的异常原因。
