# 原始需求：新建监控流程重构 (Module 1)

> **公共上下文引用**: 参见 `monitor_common_context_v1.md`

## 1. 待优化功能描述
- **功能名称**: 创建新监控任务
- **优化背景**: 
    - 目前手动创建与自动追踪入口割裂，认知成本高。
    - 批量录入效率低，缺乏统一的暂存编辑模式。
    - 自动追踪规则缺乏可见的管理界面，处于“黑盒”状态。

## 2. 核心优化需求

### 2.1 统一创建入口
- 整合“手动创建”与“自动追踪”为一个统一的“新建监控”模态框。
- 通过 Tab 分流，并在手动录入界面引导无链接用户使用自动追踪。

### 2.2 重构录入体验（暂存区模式）
- 采用纯前端“暂存表格”，支持多 URL 快速粘贴和文件导入。
- **配置继承与锁定**: 支持设置通用配置并同步到所有行，若单行被手动修改则自动锁定。
- **部分提交**: 成功项创建并移除，失败项保留并高亮错误原因。

### 2.3 自动追踪管理
- 在创建界面内直接提供对“已有追踪规则”的管理能力（查看、状态切换、编辑、删除）。
- 支持从收藏夹批量导入频道作为追踪目标。
