# 发版公告｜v7.6.0 内容监控 2.0 正式上线

尊敬的用户：

在 **v7.6.0** 版本中，我们对「内容监控」进行了整体验升级，正式上线 **内容监控 2.0**。本次更新重点围绕监控创建效率、自动追踪管理、列表查看效率，以及 **Twitter (X)** 平台支持进行了优化，帮助您更高效地管理监控任务，及时发现新内容与异常变化。

本次更新后，您可以在同一个入口完成手动创建监控和自动追踪规则配置；批量添加内容时，也可以先放入暂存区统一编辑后再提交。在监控列表中，您还能更清楚地区分任务来源、查看新任务提醒、识别临期或异常状态，并支持直接续期。同时，内容监控现已支持 Twitter (X) 内容的解析与数据展示。

## 本次更新内容

### 1. 新建监控入口整合，支持批量暂存与统一配置

- 新建监控弹窗中整合了「监控已发布内容」和「自动追踪预发布内容」两个入口
- 支持一次性粘贴多个 URL，或通过表格模板批量导入
- 批量录入的内容会先进入暂存区，便于统一设置监控时长、标签等信息后再提交
- 若您单独修改某条内容的配置，系统会保留该条自定义设置，避免被全局配置覆盖
- 批量提交时，如部分内容链接异常或额度不足，成功项会正常创建，失败项会保留并提示原因

### 2. 自动追踪规则支持可视化管理

- 在同一弹窗内即可创建和管理自动追踪规则
- 支持按「频道 + 关键词」设置自动追踪，系统会自动发现符合条件的新内容并生成监控任务
- 自动追踪产生的任务会在列表中标记来源，方便区分是手动创建还是自动生成
- 点击来源规则后，可直接回到对应规则进行查看和管理

### 3. 监控列表信息更完整，支持提醒与续期

- 列表新增来源标识，可区分手动录入与自动追踪任务
- 自动追踪生成的新任务会提供未读提示，帮助您更快发现新增内容
- 监控进度支持可视化展示，并对临期或异常停滞任务给出提醒
- 支持单条续期和批量续期，减少重复操作

### 4. 正式支持 Twitter (X) 平台监控

- 现已支持 `x.com` 与 `twitter.com` 链接的识别与监控
- 在 Twitter 视图下，系统会按平台习惯展示阅读量、转推、书签等指标
- 对于纯文字推文，系统会自动生成文字预览卡片，便于在列表中快速识别内容
- Twitter 数据会与其他平台监控数据一起纳入汇总，方便统一查看整体表现

## 如何使用

### 创建内容监控

1. 进入对应监控项目，点击「新建监控」
2. 在「监控已发布内容」中粘贴一个或多个内容链接，或导入表格模板
3. 在暂存区统一设置监控时长、标签等信息
4. 确认后点击「开始监控」

### 创建自动追踪规则

1. 进入「新建监控」
2. 切换到「自动追踪预发布内容」
3. 选择目标频道并设置关键词
4. 保存后，系统会自动追踪符合条件的新内容并生成监控任务

### 查看与管理任务

1. 在监控列表中通过平台 Tab、来源类型、状态等条件筛选任务
2. 关注新任务提示、临期提醒和异常停滞提示
3. 如需延长监控周期，可在单条任务或批量操作中选择「续期」

## 温馨提示

- 内容监控 2.0 中，手动监控和自动追踪已统一到同一个新建入口下
- 若您在新建过程中意外关闭弹窗，系统会在合适场景下提示恢复未完成的暂存内容
- 自动追踪规则按监控项目独立管理，不同项目之间不共享
- 单个监控项目中，最多可运行 10 条自动追踪规则
- Twitter 平台下会展示专属指标字段，因此和其他平台的列表列名可能略有不同
- 批量创建时，如存在无效链接或额度不足，系统会保留异常项，方便您修正后再次提交
