# 标准搜索自然语言输入 · Spec v1

> 归属版本：`v6.3.9`

## 一、信息架构

- 频道搜索在本版本只有 `标准搜索` 这一条正式搜索主线
- `自然语言` 是 `标准搜索` 内部的显式输入形态，不是一级模式
- 本版本不出现 `AI搜索（智能精选）` 入口

## 二、输入形态

标准搜索内提供：

- `关键词`
- `自然语言`

规则：

- `关键词` 继续以胶囊样式留在搜索框内
- 切到 `自然语言` 后，输入框默认空白
- 原关键词从输入框内移出，和地区、粉丝量等一起进入 `当前生效条件`

## 三、自然语言解析

用户提交自然语言后：

1. 系统轻量解析意图
2. 解析时可接收当前 active schema 作为上下文
3. 输出新的 schema 参数
4. 调用现有标准搜索 API
5. 回到标准搜索结果页

语义约束：

- 支持 `整套重建`
- 支持 `增量调整`
- 若识别为增量调整，则默认作用在当前 active schema 上

## 四、结果页壳层

自然语言成功后，顶部固定为：

1. `紧凑自然语言输入框`
2. `当前生效条件`
3. `折叠历史时间线`

约束：

- 输入框始终可用
- 历史只记录成功搜索
- 点击历史项只回填原句，不自动搜索
- 切回 `关键词` 时，自然语言历史清空
- 本版本不在结果页展示智能精选桥接 CTA

## 五、失败态

- 失败提示以内联方式贴在输入框下方
- 不做独立错误页
- 若当前已有标准搜索结果，则旧结果继续保留
- 不自动切关键词模式

## 六、验收标准

- 用户能显式切到自然语言输入
- 自然语言成功后仍是标准搜索结果页
- 顶部顺序稳定为：输入框 -> 当前生效条件 -> 折叠历史
- 失败时旧结果不丢失
- 本版本的产品 UI 中不出现智能精选入口
